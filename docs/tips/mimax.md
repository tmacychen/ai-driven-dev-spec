## coding-plan-search 使用指南

根据搜索结果和 MiniMax 官方文档，`coding-plan-search` 是 **Coding Plan MCP** 提供的核心功能之一，专门用于**联网搜索辅助编程**。

---

### 核心功能

`coding-plan-search` 为 AI Agent 提供**实时联网搜索能力**，在编程场景中解决以下问题：

| 场景 | 说明 |
|------|------|
| **查最新文档** | 查询 API 文档、框架更新的最新信息 |
| **技术调研** | 搜索开源项目、技术方案、最佳实践 |
| **解决新问题** | 查询 Stack Overflow、GitHub Issues 等解决编程问题 |
| **获取实时数据** | 查询版本号、发布日期、兼容性信息等 |

---

### 使用方式

在 MMX-CLI 中，`coding-plan-search` 主要通过以下方式使用：

#### 1. **通过 MCP Server 集成**

```bash
# 安装 MiniMax MCP Server
npx @minimax/mcp-server

# 在支持 MCP 的编辑器中使用（如 Cursor、Claude Desktop）
```

#### 2. **在 Claude Code / OpenClaw 中使用**

当你在 Claude Code 或 OpenClaw 中编程时，如果需要查询最新信息，Agent 会自动调用 `coding-plan-search`：

```
你: 如何在 React 19 中使用新的 Hook API?

Agent (自动调用 search):
→ 正在搜索 "React 19 Hook API changes 2026"
→ 搜索到官方文档、最新的迁移指南...
→ 综合最新信息给出回答
```

#### 3. **通过 MMX-CLI 直接调用**

```bash
# 使用 MMX 的搜索模式
mmx search "如何用 Python 调用 MiniMax API 2026"

# 使用编程专用模式
mmx code --search "React 19 use() hook usage example"
```

### 与普通对话的区别

| 对比项 | 普通 M 模型 | coding-plan-search |
|--------|-------------|---------------------|
| **知识截止** | 训练数据截止日期 | 实时最新 |
| **文档准确性** | 可能过时 | 基于最新官方文档 |
| **技术方案** | 通用建议 | 最新最佳实践 |
| **版本信息** | 可能不匹配 | 实时查询 |

### 注意事项

1. **配额限制**：每周 60 次对话 / 6,000 次 Token，注意合理使用
2. **重置周期**：每 3 小时 25 分钟左右重置一次配额
3. **网络依赖**：需要稳定的网络连接才能进行搜索
4. **结果时效**：搜索结果基于实时网络数据，可能因时间变化

如果你想实际体验，可以直接在 MMX-CLI 中提问需要最新信息的编程问题，Agent 会自动决定是否调用搜索功能！
