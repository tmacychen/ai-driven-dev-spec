# 应用规范示例

## 项目概述

一个简单的**待办事项管理应用** (Todo App)，支持用户注册、登录、创建/管理待办事项。

## 核心功能

1. **用户认证**
   - 用户注册（用户名、邮箱、密码）
   - 用户登录（返回 JWT Token）
   - 登出

2. **待办事项管理**
   - 创建待办事项（标题、描述、截止日期）
   - 查看待办事项列表（支持分页）
   - 标记为已完成 / 未完成
   - 编辑待办事项
   - 删除待办事项

3. **仪表板**
   - 显示待办统计（总数、已完成、进行中）
   - 最近活动时间线

## 技术栈

- **前端**: React + TypeScript
- **后端**: Node.js + Express
- **数据库**: SQLite (开发) / PostgreSQL (生产)
- **测试**: Jest + Playwright

## 验收标准

- [ ] 用户可以成功注册和登录
- [ ] 用户可以创建新的待办事项
- [ ] 用户可以标记待办事项为完成
- [ ] 用户可以查看所有待办事项
- [ ] 仪表板正确显示统计信息
- [ ] 移动端响应式布局正常
- [ ] 所有 E2E 测试通过
